## 1. What changed

The dashboard owned a v1 **CRM-style contact model** that duplicated AI-platform data and that **no read
path consumed**. Slice 12D-D removed it (rationale in ADR-0012 ¬ßContext).

| | Before (6 tables) | After (4 tables) |
|---|---|---|
| `app_tenants` | ‚úÖ | ‚úÖ |
| `app_channels` | ‚úÖ | ‚úÖ |
| `app_conversations` | ‚úÖ (+ `customer_id`, `customer_identity_id` FKs) | ‚úÖ (**columns removed**; keeps `external_contact_id`) |
| `app_tenant_entitlements` | ‚úÖ | ‚úÖ |
| `app_customers` | ‚úÖ | **DROPPED** |
| `app_customer_identities` | ‚úÖ | **DROPPED** |

The contact now lives **only** as `app_conversations.external_contact_id` (TEXT, **NOT NULL**, **indexed
not unique**), derived from `ai.agno_sessions.user_id` and **masked on read**. The AI platform
(`ai.agno_sessions.user_id` / `ai.customers`) is the contact registry of record.

## 2. The migration

`drizzle/0001_clumsy_rawhide_kid.sql` (generated by `drizzle-kit generate`, then **hardened**):

- `ALTER TABLE ‚Ä¶ DISABLE ROW LEVEL SECURITY` √ó2 (drizzle pre-drop step)
- `DROP TABLE "dashboard"."app_customer_identities" CASCADE;`
- `DROP TABLE "dashboard"."app_customers" CASCADE;`
- `DROP CONSTRAINT IF EXISTS` √ó2 ‚ÄĒ the two `app_conversations` FK constraints. **Hardened with `IF
  EXISTS`** because the preceding `DROP TABLE ‚Ä¶ CASCADE` already removes them; without `IF EXISTS` the
  explicit drops would error ("constraint does not exist") on apply.
- `DROP INDEX IF EXISTS "app_conv_customer_idx";` / `"app_conv_identity_idx";`
- `DROP COLUMN IF EXISTS "customer_id";` / `"customer_identity_id";`

**DDL-only.** No `INSERT`. No reference to `ai.*`. Locked by `lib/db/migration.test.ts` (kept-table set +
DROP-migration assertions + the long-standing "never touches `ai.*` / no transcript / no seed" guards).

## 6. Rollback

`ai.*` is unaffected, so rollback is dashboard-only and low-risk:

```bash
psql "$DATABASE_URL" -c 'DROP SCHEMA dashboard CASCADE;'
psql "$DATABASE_URL" -f backups/2026-06-16-dashboard-pre-12dd.sql
```

This restores the exact pre-migration 6-table state (DDL + data). Drizzle migrations are forward-only;
there is no down-migration (intentional ‚ÄĒ the backup is the rollback source).
