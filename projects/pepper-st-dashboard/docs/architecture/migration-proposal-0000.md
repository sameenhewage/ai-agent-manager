## Summary

Creates the dashboard-owned `dashboard` schema and its **6** `app_*` tables (Drizzle ORM
→ generated SQL). DDL only: `CREATE SCHEMA` + 6 `CREATE TABLE` + 10 FK `ALTER TABLE`
(idempotent) + 8 `CREATE INDEX`. **No seed/INSERT**, **no `ai.*` DDL**, **no FK into
`ai.*`**. Multi-tenant from day one (`tenant_id` on every operational table).

### Benign drizzle-kit conventions (not deviations)

- `CREATE TABLE IF NOT EXISTS` and FKs added via idempotent `DO $$ … WHEN
  duplicate_object THEN null` blocks (drizzle-kit default) — semantically equal to the
  proposal's inline `REFERENCES`.
- FK constraints are **named** (e.g. `app_conversations_tenant_id_app_tenants_id_fk`)
  rather than anonymous inline — same effect.
- Descending index emitted as `("tenant_id","last_at" DESC NULLS LAST)` — matches intent
  of `(tenant_id, last_at DESC)`.
- Column defaults `gen_random_uuid()` / `now()` / `'active'` / `'pending'` /
  `'Asia/Colombo'` / `'open'` / `true` match the proposal.
